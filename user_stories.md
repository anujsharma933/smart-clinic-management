User Story 1: Doctor Login

Title: Doctor Login to System

Description: As a doctor, I want to log in to the system using my credentials so that I can securely access my dashboard.

Acceptance Criteria:

Given valid credentials, when the doctor logs in, then access should be granted.

Given invalid credentials, when the doctor logs in, then an error message should be displayed.

Priority: High

Story Points: 3

User Story 2: View Available Time Slots

Title: View Available Time Slots

Description: As a doctor, I want to view my available time slots so that I can manage my schedule effectively.

Acceptance Criteria:

The doctor should be able to view all available time slots.

The displayed slots should reflect the current schedule accurately.

Priority: Medium

Story Points: 5

User Story 3: View Patient Appointments

Title: View Patient Appointments

Description: As a doctor, I want to view my scheduled patient appointments so that I can prepare for consultations.

Acceptance Criteria:

The doctor should be able to view a list of appointments.

Each appointment should show patient name, date, and time.

Priority: High

Story Points: 5
